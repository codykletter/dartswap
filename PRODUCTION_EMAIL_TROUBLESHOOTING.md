# Production Email Troubleshooting Guide

## 1. Problem Overview
You may experience a situation where user **registration succeeds, but no confirmation email is received** by the user. The user account is successfully created in the system, but the welcome/verification email never arrives in their inbox.

## 2. Root Cause Analysis
This behavior occurs due to a **graceful degradation** pattern implemented in the codebase. 

Specifically:
- Users are created in the MongoDB database *before* the email sending process begins.
- The registration code (`app/api/auth/register/route.ts`) wraps the email sending process in a `try/catch` block. If the email fails to send, the system catches the error, logs it, and still returns a successful registration response to the user.
- The email utility (`lib/email.ts`) is designed to fail silently and return `null` if critical configuration variables like `EMAIL_USER` or `EMAIL_PASS` are missing.

Therefore, if users are not appearing in your database at all, it is a **database issue**, not an email issue. If users are in the database but emails aren't arriving, it is an **email configuration issue**.

## 3. Step-by-Step Diagnostic Process

### A. Access Netlify Function Logs
1. Log in to your Netlify dashboard.
2. Select your site.
3. Go to **Logs** > **Functions**.
4. Look for the `api/auth/register` or `api/auth/verify-email` function logs.

### B. Look for Specific Log Patterns
Search the logs for the exact debug output format generated by the system. You will typically see:
- `[EMAIL DEBUG] ...`
- `EMAIL DEBUG INFO:` followed by configuration status (e.g., whether `EMAIL_USER` is defined).
- `EMAIL SEND RESULT:` which will indicate success or the specific error message.

### C. Verify Environment Variables in Netlify
1. In the Netlify dashboard, go to **Site configuration** > **Environment variables**.
2. Verify that all 9 required variables exist and have correct values (see Quick Fix Checklist below).

### D. Check Brevo Configuration
1. Log in to your [Brevo Dashboard](https://app.brevo.com).
2. **Sender Verification:** Go to **Senders, Domains & Dedicated IPs** > **Senders** and verify your sender email is active and verified.
3. **Daily Limits:** Check your main dashboard to ensure you haven't exceeded the 300 emails/day limit on the free tier.
4. **SMTP Credentials:** Go to **Your Profile Name** (top right) > **SMTP & API** > **SMTP** tab to verify your credentials.

### E. Verify MongoDB Connection
1. Log in to [MongoDB Atlas](https://cloud.mongodb.com).
2. Go to **Database** > **Browse Collections**.
3. Check the `users` collection to confirm if the newly registered test user exists.

## 4. Quick Fix Checklist

If emails aren't sending, immediately perform these actions in order:

1. **Access Netlify Variables:** Go to Netlify dashboard → Site configuration → Environment variables.
2. **Verify Required Variables:** Ensure these 9 variables are set correctly:
   - `EMAIL_HOST` (e.g., smtp-relay.brevo.com)
   - `EMAIL_PORT` (e.g., 587)
   - `EMAIL_USER` (Your Brevo SMTP login)
   - `EMAIL_PASS` (Your Brevo SMTP master password)
   - `EMAIL_FROM` (e.g., noreply@yourdomain.com)
   - `MONGODB_URI`
   - `JWT_SECRET`
   - `NODE_ENV` (should be "production")
   - `NEXT_PUBLIC_APP_URL` (Your production URL, e.g., https://your-site.netlify.app)
3. **Get Brevo Credentials:** Retrieve correct SMTP credentials from Brevo dashboard → Profile → SMTP & API.
4. **Verify Sender:** Ensure `EMAIL_FROM` exactly matches a verified sender in Brevo dashboard → Senders.
5. **Redeploy:** Redeploy the site in Netlify after adding or updating any variables (Deploys → Trigger deploy → Deploy site).
6. **Test:** Test registration and check function logs again.

## 5. Common Issues & Solutions

- **Missing `EMAIL_USER` or `EMAIL_PASS` (80% Probability):** The variables were either not copied over to Netlify or have a typo in the key name.
  * **Solution:** Add them exactly as formatted in the Quick Fix Checklist.
- **Wrong Brevo Credentials:** Using your Brevo *account login password* instead of the *SMTP password*.
  * **Solution:** Generate or copy the correct SMTP password from the Brevo SMTP & API settings page.
- **Unverified Sender Email in Brevo:** Brevo will reject emails sent from an address it hasn't verified.
  * **Solution:** Verify the sender email address in Brevo's Senders configuration.
- **Brevo Daily Limit Reached:** The free tier only allows 300 emails per day.
  * **Solution:** Check usage in Brevo. If reached, you must wait until the next day or upgrade your plan.
- **MongoDB Connection Issues:** If users aren't appearing in the database at all.
  * **Solution:** Check `MONGODB_URI` and ensure Netlify IPs (or `0.0.0.0/0` for all IPs) are whitelisted in MongoDB Atlas Network Access.

## 6. How to Verify the Fix

1. **Register a Test Account:** Go to your production site and register a brand new account.
2. **Check Netlify Logs:** Look at the function logs for the "EMAIL DEBUG INFO" section and the "EMAIL SEND RESULT" success message.
3. **Check Inbox:** Verify the email arrived in the test account's inbox (or spam/junk folder).
4. **Verify Database:** Confirm the user appears in your MongoDB Atlas database.

## 7. Additional Debugging

- **Using the Test Script Locally:** If you are unsure if your Brevo credentials work at all, run `npx ts-node scripts/test-email.ts` locally with your `.env.local` populated. This isolates email testing from the registration logic.
- **Checking MongoDB Atlas Network Access:** Go to MongoDB Atlas → Security → Network Access. Ensure you have an entry for `0.0.0.0/0` (Allow Access From Anywhere) so Netlify's dynamic IPs can connect.
- **Verifying Production URL:** Ensure `NEXT_PUBLIC_APP_URL` does not have a trailing slash (e.g., `https://dartswap.netlify.app`, not `https://dartswap.netlify.app/`) as this is used to construct the verification link in the email.
